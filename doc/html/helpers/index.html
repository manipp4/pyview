This section contains the documentation of the helper classes.

Data Manager¶

class pyview.helpers.datamanager.DataManager¶

Bases: pyview.lib.patterns.Singleton, pyview.lib.patterns.Reloadable, pyview.lib.patterns.ThreadedDispatcher, pyview.lib.patterns.Subject, pyview.lib.patterns.Observer

The DataManager is a Singleton class which can be used to keep track of datacubes. Just call addDatacube() to add a datacube to the DataManager. pyview.gui.datamanager.DataManager provides a graphical frontend of the DataManager class.

addDatacube(datacube)¶

Adds a datacube to the data manager.

autoPlot(datacube, clear=False)¶

Triggers an autoPlot of the selected datacube, i.e. a graphical frontend that observeres the state of the data manager will be notified that ‘datacube’ should be replotted.

clear()¶

Removes all datacubes from the data manager.

datacubes()¶

Returns the datacubes.

removeDatacube(datacube)¶

Removes a datacube from the data manager.

Code Runner¶

class pyview.helpers.coderunner.CodeRunner(gv={}, lv={})¶

Bases: pyview.lib.patterns.Reloadable, pyview.lib.patterns.Subject

A class that manages the execution of code in different threads.

clear(gv={}, lv={})¶

Reinitializes the class and deletes all running threads.

clearExceptions()¶

Clears all exceptions that are stored in the exception dictionary.

currentWorkingDirectory()¶

Returns the current working directory.

executeCode(code, identifier, filename=None, lv=None, gv=None)¶

Executes a code string with a given identifier, filename and local and global variable dictionary.

formatException(identifier)¶

Returns a formatted exception string for the given identifier, or an empty string if no exception is available.

getException(identifier)¶

Returns the exception thrown by the code thread with the given identifier, or None if no exception is present.

getId()¶

Returns a unique ID which can be used to identify a code thread.

getTraceback(identifier)¶

Returns the traceback thrown by the code thread with the given identifier, or None if no traceback is present.

gv()¶

Returns the global variables dictionary.

hasFailed(identifier)¶

Returns True if the thread with the given identifier has terminated abnormally.

isExecutingCode(identifier=None)¶

Returns True if the thread with the given identifier is currently executing code.

lv()¶

Returns the local variables dictionary.

setCurrentWorkingDirectory(directory)¶

Changes the current working directoy.

status()¶

Returns a dictionary containing information on all threads that are managed by the code runner.

stopExecution(identifier)¶

Stops the execution of code in the thread with the given identifier by asynchronously raising a special exception in the thread.
